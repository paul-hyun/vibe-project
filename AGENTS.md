# AGENTS.md

이 문서는 이 프로젝트에서 AI 코딩 어시스턴트가 따라야 하는 기본 규칙이다.
목표는 일관된 기술 스택과 안전한 개발 습관을 유지하는 것이다.

## 프로젝트 기본 스택

- 프레임워크: Next.js (App Router)
- 언어: TypeScript
- 스타일: Tailwind CSS
- 데이터베이스/백엔드: Supabase

## DO (반드시 지켜야 할 것)

- Next.js는 `app` 디렉터리 기반의 App Router 방식으로 작성한다.
- 페이지, 레이아웃, 로딩/에러 처리는 App Router 규칙(`page.tsx`, `layout.tsx` 등)을 따른다.
- 모든 새 코드(컴포넌트, 유틸, 서버 로직)는 TypeScript로 작성한다.
- 타입 안정성을 우선하고, 가능한 한 명시적인 타입을 사용한다.
- 스타일링은 Tailwind CSS 유틸리티 클래스를 기본으로 사용한다.
- 데이터 저장/인증/조회는 Supabase를 기준으로 구현한다.
- 환경 변수는 `.env` 계열 파일로 관리하고, 코드에는 `process.env`를 통해 참조한다.
- 서버에서 처리해야 하는 로직(비밀키 필요 작업, 민감 데이터 접근)은 서버 전용 코드로 분리한다.
- 코드 제안 시 폴더 구조와 파일 이름을 Next.js 관례에 맞춘다.
- 초보 개발자가 이해하기 쉽게, 복잡한 구현에는 짧은 설명 주석을 함께 제공한다.

## DON'T (하면 안 되는 것)

- Pages Router 방식(`pages` 라우팅)을 기본 전제로 코드를 작성하지 않는다.
- JavaScript 전용 코드로 작성하지 않는다. (`.js` 대신 `.ts`/`.tsx` 우선)
- 인라인 스타일/CSS 파일을 기본 선택으로 남발하지 않는다. (Tailwind 우선)
- 데이터베이스를 임의로 다른 서비스로 가정하지 않는다. (기본 DB는 Supabase)
- Supabase 서비스 키, API 키, 토큰, 비밀번호 등 민감 정보를 코드에 하드코딩하지 않는다.
- 클라이언트 컴포넌트에 비밀 정보나 서버 전용 로직을 넣지 않는다.
- 타입 에러를 `any` 남용이나 무리한 타입 단언으로 숨기지 않는다.
- 사용자가 요청하지 않은 대규모 구조 변경(폴더 이동, 전면 리팩터링)을 임의로 진행하지 않는다.
- 에러 처리 없이 네트워크/DB 호출 코드를 작성하지 않는다.
- 접근성(버튼 역할, 폼 라벨 등)과 기본 UX 피드백(로딩/에러 상태)을 생략하지 않는다.

## 작업 원칙

- 변경 범위는 요청 사항에 맞게 최소화한다.
- 새 파일을 만들 때는 왜 필요한지 짧게 설명한다.
- 실행/검증 방법(예: dev 서버 실행, 체크할 페이지)을 함께 안내한다.
- 불확실한 요구사항은 추정으로 단정하지 말고 확인 질문을 먼저 한다.
